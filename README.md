#### Learning C++ 
- 🌱 I’m currently focused on learning programming through C++. In January of 2025 I started learning C++ through a few books. I [started with](https://github.com/nantr0nic/cpp-learning/tree/main/bookwork/bcpp-game) "Beginning C++ Through Game Programming" by Michael Dawson. After that, I [went through](https://github.com/nantr0nic/cpp-learning/tree/main/bookwork/ppp3) Bjarne Stroustrup's "Programming: Principles and Practice Using C++" (3rd edition). One of my favorite studies from that book was [a deep dive](https://github.com/nantr0nic/cpp-learning/blob/main/bookwork/ppp3/deep_dive.md) into construction/destruction of objects, memory allocation, and compiler optimizations of the "essential operations". I am continuing to learn C++ techniques and best practices from "Modern C++ Programming Cookbook" (Marius Bancila) and supplement Bjarne's excellent books (PPP and "A Tour of C++") with "Beginning C++23" by Horton and Weert. 
#### C++ Project Log
- 🎮 In August/September of 2025, I created a project called [PadCast](https://www.github.com/nantr0nic/PadCast) - a lightweight gamepad visualization tool for streamers built with C++.
- 🎮 In October of 2025, I started to study/practice C++ Game Dev and learn about architectural patterns.
- 🎮 In November/December of 2025, I started an [SFML 3 Game Template Project](https://github.com/nantr0nic/sfml3-game-template/) to practice game dev concepts and design patterns.
- 🎮 In December of 2025, I created a simple game called [Breakdown](https://github.com/nantr0nic/breakdown) using the SFML 3 Game Template. Breakdown was a way to practice using ECS architecture, use a "data-driven" design that loads game configuration data during runtime from TOML files, and extending certain facilities like a logger, config manager, resource manager, and so on. I also got to practice creating GUI elements via an entity factory using EnTT and SFML.
